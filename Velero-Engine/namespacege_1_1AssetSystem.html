Detailed Description

System to load and access to assets.

Assets are identified by its path, be it absolute or relative. They can be used interchangeably. For example, if you load "/project/absolute/path/assets/images/player.png" you can then access it just by "assets/images/player.png". Also, this system uses a list of possible relative paths, so if you add, lets say, "assets/images" to the list, you would be able to load and get the same asset just with "player.png".
